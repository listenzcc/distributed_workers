角色

简称 角色
UI 用户交互界面软件
BCI 脑机交互软件
GAMER 游戏软件

通信表

模式 通信方向 发送包
离线 UI -> BCI 心跳包，开始离线采集包，结束离线采集包，建模包
离线 BCI -> UI 心跳包，模型准确率包，运行时错误包
在线 UI -> BCI 心跳包，开始在线采集包，结束在线采集包
在线 BCI -> UI 心跳包，运行时错误包
在线 GAMER -> BCI 心跳包，查询包
在线 BCI -> GAMER 心跳包，查询结果包，运行时错误包

所有通信包分为发送包、回复包和运行时错误包三类。
收到发送包或运行时错误包应立即回复回复包；
收到回复包无需回复。

典型通信过程：

心跳包

  1. A -> B 心跳包

  2. B -> A 回复心跳包

    (其中，A、B 为任意角色)

离线游戏中，UI 要求 BCI 开始采集

  1. UI -> BCI 开始离线采集包
  2. BCI -> UI 回复包

离线游戏中，UI 要求 BCI 进行建模

  1. UI -> BCI 建模包
  2. BCI -> UI 回复包
  3. BCI -> UI 模型准确率包
  4. UI -> BCI 回复包

在线游戏中，GAMER 要求 BCI 估计标签

  1. GAMER -> BCI 查询包
  2. BCI -> GAMER 回复包
  3. BCI -> GAMER 查询结果包
  4. GAMER -> BCI 回复包

BCI 向其他人发送运行时错误

  1. BCI -> Other 运行时错误包

  2. Other -> BCI 回复包

    (其中，Other 为 Gamer 或 UI)

发送包

心跳包

所有通信参与者都可以定期发送心跳包。

{
  "mode": "keepalive",
  "timestamp": "1585297645.123"
}

开始离线采集包

{
  "mode": "Offline",
  "cmd": "kaishicaiji",
  "subjectid": "name-001",
  "sessionid": "motion-001",
  "timestamp": "1585297645.123"
}

结束离线采集包

{
  "mode": "Offline",
  "cmd": "jieshucaiji",
  "timestamp": "1585297645.123"
}

建模包

{
  "mode": "Offline",
  "cmd": "jianmo",
  "shujulujing": "\\folder1\\folder2\\datafolder\\",
  "timestamp": "1585297645.123"
}

模型准确率包

{
  "mode": "Offline",
  "cmd": "zhunquelv",
  "moxinglujing": "\\folder1\\folder2\\modefile",
  "zhunquelv": "0.95",
  "timestamp": "1585297645.123"
}

开始在线采集包

{
  "mode": "Online",
  "cmd": "kaishicaiji",
  "moxinglujing": "\\folder1\\folder2\\modefile",
  "timestamp": "1585297645.123"
}

结束在线采集包

{
  "mode": "Online",
  "cmd": "jieshucaiji",
  "timestamp": "1585297645.123"
}

查询包

{
  "mode": "Query",
  "duration": "3.0",
  "timestamp": "1585297645.123"
}

查询结果包

{
  "mode": "QueryReply",
  "label": "1",
  "timestamp": "1585297645.123"
}

回复包

回复心跳包

{
  "mode": "Reply",
  "state": "keepalive",
  "timestamp": "1585297645.123"
}

回复能够被正确识别的包

代表开始执行所需操作。

{
  "mode": "Reply",
  "state": "OK",
  "timestamp": "1585297645.123"
}

回复无法被正确识别的包

代表无法执行任何操作。

{
  "mode": "Reply",
  "state": "ParseError",
  "timestamp": "1585297645.123"
}

运行时错误包

代表执行时遇到错误。
在 detail 字段中，应含有简要描述。

状态错误

代表运行状态错误，如重复开始采集、重复停止采集等。

{
  "mode": "RuntimeError",
  "type": "StateError",
  "detail": "xxxx",
  "timestamp": "1585297645.123"
}

文件错误

代表目标文件无法获取，如无法找到、无法读取等。

{
  "mode": "RuntimeError",
  "type": "FileError",
  "detail": "xxxx",
  "timestamp": "1585297645.123"
}

资源忙错误

代表后台资源忙，操作无法运行。

{
  "mode": "RuntimeError",
  "type": "BusyError",
  "detail": "xxxx",
  "timestamp": "1585297645.123"
}

未定义错误

代表无法事先定义，但运行时发生的错误，如内存满、关键设备断开等。

{
  "mode": "RuntimeError",
  "type": "UnknownError",
  "detail": "xxxx",
  "timestamp": "1585297645.123"
}
